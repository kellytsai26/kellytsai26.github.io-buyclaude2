面向三：核心功能 - Skill

Skill = AI 的「外掛知識包」或「專業手冊」

模式一：知識注入

將團隊的規範、風格指南、領域知識寫成 Skill，讓 Claude 在每次對話中自動套用。例如：將公司的 API 設計規範、品牌文案風格、程式碼命名規則存成 Skill。

模式二：任務自動化

將重複性高、格式固定的任務定義成 Skill。例如：每週自動產出 KPI 報告、費用報表，並依指定規則完成。

模式三：腳本執行

以程式或流程腳本方式，讓 Claude 具備超級提示能力。例如：一鍵產出互動式教學頁，或自動產生 API 文件、HTML 視覺化報告。

模式四：團隊標準化

將公司常用語言、審核規範、測試覆蓋率要求寫成 Skill。讓程式碼審查、行銷文案、資料清理皆可自動套用團隊規範。

模式五：跨工具整合

結合 Gmail、Google Drive、Calendar、Sheets 等。實現讀取 Gmail → 更新 Sheets → 建立 Calendar 事件，一次完成完整流程。

補充說明：
Skill 類似「一段被 Claude 在特定情境自動執行的 SOP」， Agent 則為可自主規劃與執行下一步的進階程式。
